REPLIT MASTER PROMPT — GigAid Admin Backend (Founder Control Panel)
Objective

Build a complete, production-ready Admin Backend Interface for GigAid that allows founders/admins to:

Manage users

Manage billing (Stripe)

View user behavior & analytics

Integrate and control Customer.io

Operate GigAid without touching production databases

🚨 HARD CONSTRAINTS (NON-NEGOTIABLE)

❌ Do NOT change any existing GigAid user-facing features, flows, UI, or behavior

❌ Do NOT modify onboarding, booking, reminders, jobs, or pricing logic

❌ Do NOT reuse customer UI components

❌ Do NOT introduce new UI or logic visible to end users

✅ Admin system must be completely isolated

✅ Backend-first implementation

✅ All admin actions must be auditable

Architecture Requirements
Admin Surface

Admin lives under /admin or admin.gigaid.ai

All APIs namespaced under /api/admin/*

Admin routes blocked server-side for non-admin users

Data Safety

Read + Act model only

Admin may update user state, billing state, and integrations

Admin may NOT rewire application logic

Destructive actions require confirmation

Prefer reversible actions where possible
