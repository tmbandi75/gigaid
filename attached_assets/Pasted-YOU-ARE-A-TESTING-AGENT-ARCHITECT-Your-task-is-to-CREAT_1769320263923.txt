YOU ARE A TESTING AGENT ARCHITECT.

Your task is to CREATE and REGISTER a set of READ-ONLY evaluation agents for a mobile + web application.

These agents:
- MUST NOT modify code
- MUST NOT write tests
- MUST NOT change config
- MUST NOT mutate state
- MUST ONLY evaluate, inspect, and report

Each agent must:
- Have a clear scope
- Have explicit PASS / FAIL criteria
- Produce deterministic output
- Be safe to run automatically

────────────────────────────────────────
GLOBAL AGENT RULES (APPLY TO ALL)
────────────────────────────────────────
- Read-only inspection only
- No code generation
- No refactors
- No suggestions
- No speculative language
- No fixes
- No retries
- If uncertain → FAIL

Each agent returns:
- Status: PASS / FAIL
- Failures: list (if any)
- Evidence: file, screen, or logic reference

────────────────────────────────────────
AGENT DEFINITIONS
────────────────────────────────────────

────────────────────
1️⃣ AuthRegressionAgent
────────────────────
Purpose:
Prevent auth regressions across web and mobile.

Checks:
- Web login (Replit Auth) functions
- Mobile login (Firebase) functions
- /api/me resolves with:
  - Replit session
  - App JWT
- Invalid Firebase token rejected
- App JWT expiry handled gracefully

Triggers:
- Auth middleware
- Headers
- Cookies
- Sessions
- User table

Fail if:
- Any auth path is broken or ambiguous

────────────────────
2️⃣ AppleReviewComplianceAgent
────────────────────
Purpose:
Prevent App Store rejection.

Checks:
- Apple Sign In exists
- Apple button is FIRST
- Email or Phone fallback exists
- Terms + Privacy visible pre-login
- SMS disclosure present
- “Skip for now” never appears pre-auth
- No login method hidden

Triggers:
- Any auth UI change

Fail if:
- Any Apple guideline violated

────────────────────
3️⃣ AccountLinkingAgent
────────────────────
Purpose:
Prevent duplicate or orphaned users.

Checks:
- Existing Replit user logging in via mobile resolves to same user_id
- Firebase login with same email links correctly
- Phone login does NOT create duplicates
- emailNormalized populated
- auth_provider not overwritten

Triggers:
- User creation
- Auth logic changes

Fail if:
- Duplicate user paths exist

────────────────────
4️⃣ SessionPersistenceAgent
────────────────────
Purpose:
Catch real-world mobile auth failures.

Checks:
- Kill app → reopen → session valid
- Background → foreground → session valid
- Expired session leads to clean re-login
- No infinite auth loops

Fail if:
- Unexpected logout
- Looping behavior

────────────────────
5️⃣ PaymentsGateAgent
────────────────────
Purpose:
Ensure auth never blocks money flows.

Checks:
- Auth does NOT block:
  - Viewing jobs
  - Completing jobs
  - Viewing payouts
- Step-up auth not triggered accidentally
- Read-only flows work near expiry

Fail if:
- Revenue paths blocked by auth

────────────────────
6️⃣ AuthCopyAgent
────────────────────
Purpose:
Ensure auth copy is reviewer-safe.

Checks:
- No raw errors shown
- Errors are human-readable
- No dev/debug copy
- No technical jargon (JWT, Firebase, token)

Fail if:
- Developer-facing language exposed

────────────────────
7️⃣ FirstRunAgent
────────────────────
Purpose:
Validate first-time user experience.

Checks:
- Fresh install → welcome screen
- First login → dashboard
- Optional setup is skippable
- Skip never blocks usage

Fail if:
- First-run dead ends exist

────────────────────
8️⃣ ReleaseDriftAgent
────────────────────
Purpose:
Prevent last-minute UI drift.

Checks:
- Auth UX matches locked spec
- Button order unchanged
- Copy unchanged
- Legal links present

Triggers:
- Release branch
- Pre-TestFlight

Fail if:
- Any drift detected

────────────────────
9️⃣ FirstDollarAgent
────────────────────
Purpose:
Protect “first win” moment.

Checks:
- New user can:
  - Sign in
  - Create first job
  - Accept booking
- Auth never blocks money flow
- Errors readable on expiry

Fail if:
- First revenue path blocked

────────────────────
🔟 UpgradeFlowAgent
────────────────────
Purpose:
Protect plan upgrades.

Checks:
- Session persists through upgrade
- No re-login required
- Session survives payment redirect

Fail if:
- Upgrade interrupted by auth

────────────────────
1️⃣1️⃣ PrivacyComplianceAgent
────────────────────
Purpose:
Match Apple privacy declarations.

Checks:
- Only declared data collected
- No logging of:
  - Emails
  - Phones
  - Tokens
- No silent SDK data collection on auth screens

Fail if:
- Privacy mismatch detected

────────────────────
1️⃣2️⃣ AuthResilienceAgent
────────────────────
Purpose:
Catch auth crash paths.

Checks:
- Apple sheet dismissal handled
- SMS delay handled
- Network drop handled
- Loading states always resolve

Fail if:
- Silent failure or crash possible

────────────────────────────────────────
REGISTRATION
────────────────────────────────────────
Register all agents as:
- Independent evaluators
- Callable individually
- Callable via ReleaseGateAgent orchestration

Agents MUST NOT auto-run unless invoked by:
- Developer
- ReleaseGateAgent
- Orchestrator

────────────────────────────────────────
OUTPUT STANDARD (ALL AGENTS)
────────────────────────────────────────
Return:
- Agent name
- Status: PASS / FAIL
- Failures (if any)
- Evidence reference

NO recommendations.
NO fixes.
NO next steps.

END.
