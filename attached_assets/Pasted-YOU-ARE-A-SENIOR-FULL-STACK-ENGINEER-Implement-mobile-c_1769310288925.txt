STEP 3 — Add backend endpoints (additive)
Add:
A) POST /api/auth/mobile/firebase
Request: { idToken: string }
Behavior:
  1) Verify idToken with Firebase Admin.
  2) Extract firebase_uid = decoded.uid
  3) Extract email (decoded.email) if present; normalize to email_normalized
  4) Extract phone number (decoded.phone_number) if present; normalize to E.164 already provided by Firebase
  5) Find existing user in this order:
     - If email_normalized present: find user where email_normalized matches
     - Else if phone_e164 present: find user where phone_e164 matches
     - Else: create a new user (edge case; allow but warn)
  6) Link identity:
     - Set auth_provider='firebase' and auth_provider_id=firebase_uid if empty OR if the user currently has replit and we allow multiple identities:
         Option 1 (simpler): keep columns for "primary provider" only:
           - If user exists with Replit auth, DO NOT overwrite auth_provider/auth_provider_id.
           - Instead add table auth_identities(user_id, provider, provider_id, created_at) and insert firebase identity.
         Option 2 (acceptable quick path): add firebase_uid column separate:
           - firebase_uid TEXT NULL unique
           - Keep replit columns as-is.
     Choose the least invasive approach based on existing schema.
  7) Update user.email/email_normalized if empty and Firebase provided email.
  8) Update user.phone_e164 if empty and Firebase provided phone.
  9) Issue backend session JWT (APP session):
     - payload: { sub: user_id, provider: 'firebase', firebase_uid, email_normalized }
     - exp: 30 days
  10) Return { token: appJwt, user: { id, email, phone } }

B) GET /api/me

  - existing Replit session/cookie for web
- Return the same user shape for both.
