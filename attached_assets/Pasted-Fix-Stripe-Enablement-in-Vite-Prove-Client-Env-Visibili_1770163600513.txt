Objective

Resolve the issue where Stripe Checkout never opens because the client bundle does not see VITE_STRIPE_ENABLED.

This prompt will:

ROOT CAUSE (CONFIRMED)

The browser runtime does not have access to:

import.meta.env.VITE_STRIPE_ENABLED


Therefore:

STRIPE_ENABLED === false


This is not caused by CSP warnings, Firebase, Stripe setup, or UI logic.

In Vite apps, only env vars prefixed with VITE_ and defined in Replit Secrets are exposed to the client bundle at build time.

1️⃣ Add a Client Env Probe (MANDATORY)

Create this file:

client/src/debug/envProbe.ts

export function logClientEnv() {
  console.log('🧪 CLIENT ENV PROBE', {
    VITE_STRIPE_ENABLED: import.meta.env.VITE_STRIPE_ENABLED,
    ENV_KEYS: Object.keys(import.meta.env)
  })
}

2️⃣ Call the Probe at App Startup

In App.tsx (or the root authenticated app component), add:

Open:

client/src/lib/stripeCheckout.ts


Replace all Stripe enablement logic with:

export const STRIPE_ENABLED =
  import.meta.env.VITE_STRIPE_ENABLED === 'true'

console.log('[Stripe env check]', {
  VITE_STRIPE_ENABLED: import.meta.env.VITE_STRIPE_ENABLED,
  STRIPE_ENABLED
})

export async function startStripeCheckout(plan: string) {
  if (!STRIPE_ENABLED) {
    console.error(
      '[Stripe] Checkout disabled — VITE_STRIPE_ENABLED is missing or false'
    )

    alert(
      'Stripe is disabled in this environment.\n\n' +
      'Fix: Add VITE_STRIPE_ENABLED=true to Replit Secrets and restart the app.'
    )

    throw new Error('Stripe disabled by env guard')
  }

  // Existing checkout logic continues unchanged
}
