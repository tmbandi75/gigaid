## GLOBAL RULES (SOURCE OF TRUTH)

### Definitions

* **Delete** = permanent removal
* **Archive** = hidden from active views, retained for history

## OBJECT-LEVEL RULES

### 🧾 Invoices

#### Allowed Actions

* **Archive** → always
* **Delete** → ONLY if:

  * status = `draft` OR `not_sent`
  * not paid
  * no Stripe activity

#### Forbidden

* Delete if:

  * status = `paid`
  * status = `sent`
  * Stripe payment exists

#### UI Behavior

Swipe LEFT → reveal:

* Archive
* Delete (only if eligible)

#### Backend Guard

Reject delete if conditions fail.

---

### 🧰 Jobs

#### Allowed Actions

* **Archive** → completed or canceled jobs
* **Cancel Job** → upcoming jobs

#### Delete Allowed ONLY if:

* status = `draft`
* no client notifications sent
* no invoice attached

#### Forbidden

* Delete in-progress jobs
* Delete completed jobs

#### UX Language

* Prefer **Cancel** or **Archive**
* Avoid “Delete Job” except for drafts

---

### 👥 Requests / Leads

#### Allowed Actions

* **Delete** → always
* **Archive** → optional (secondary)

#### UX

* Swipe LEFT → Delete
* Confirmation optional (allowed to skip)

---

## DRAFT OBJECTS (SAFE DELETE)

* Draft Jobs → delete allowed
* Draft Invoices → delete allowed
