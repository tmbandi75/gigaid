Canonical Plan Model
type PlanTier = 'free' | 'pro' | 'pro_plus' | 'business'

const PLAN_ORDER = {
  free: 0,
  pro: 1,
  pro_plus: 2,
  business: 3
}

REQUIRED FIXES
1️⃣ Subscription State Handling (MANDATORY)

Use one subscription hook:

const {
  plan: currentPlan,
  hasSubscription,
  isLoading,
  isError,
  userId
} = useSubscription()


Rules:

plan must come from backend

Never default to free

Never block UI on error

Loading state must resolve

2️⃣ Loading State Rules (CRITICAL)
❌ Forbidden
const isWaitingForData = isLoading || !subscription

✅ Correct
const isWaitingForData = isLoading


If loading fails, buttons must still work.

3️⃣ Button Disabled Logic (Authoritative)

Buttons may ONLY be disabled when:

function isPlanDisabled(target: PlanTier, current: PlanTier, isLoading: boolean) {
  if (isLoading) return true
  if (current !== 'free' && target === current) return true
  return false
}


Nothing else may disable a button.

4️⃣ Upgrade Detection
function isUpgrade(target: PlanTier, current: PlanTier) {
  return PLAN_ORDER[target] > PLAN_ORDER[current]
}

5️⃣ Click Handling (CLICK-TIME DECISION — NOT RENDER-TIME)
async function handlePlanClick(targetPlan: PlanTier) {
  console.log('[pricing click]', {
    userId,
    currentPlan,
    targetPlan,
    isLoading
  })

  if (isLoading) return

  if (isUpgrade(targetPlan, currentPlan)) {
    startStripeCheckout(targetPlan)
    return
  }

  router.push('/settings/billing')
}


This guarantees:

Clicks always do something

Stripe is called for upgrades

Downgrades are routed correctly

Failures are visible

6️⃣ Button Labels (Required)
function getButtonLabel(target: PlanTier, current: PlanTier) {
  if (target === current) return 'Current Plan'
  if (isUpgrade(target, current)) return 'Upgrade'
  return 'Manage Subscription'
}
