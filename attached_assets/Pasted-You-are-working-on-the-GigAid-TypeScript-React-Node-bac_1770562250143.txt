B) UTM + REFERRER CAPTURE (CLIENT + SERVER)
==================================================

1) Client-side UTM capture
- On first visit to any public page (homepage + booking pages + public invoice/review), parse URL query params.
- Store UTM + landing_path + referrer_code in localStorage with TTL 14 days.
- If user signs up/logs in, POST attribution to server.

2) Server-side attribution persistence
Create:
POST /api/attribution/track
Body: { landingPath, source?, utm_*, referrerCode? }

Rules:
- If acquisition_attribution already exists: update last_touch + increment touch_count, but do not overwrite first_touch.
- If referrerCode is present: resolve to referrals.referrer_user_id and set referrer_user_id.
- Enforce per-user rate limit (e.g. max 20 touches/day) to avoid spam.

Analytics event:
posthog: acquisition_touch_recorded
