## 1️⃣ Webhook: Update User Plan on Subscription Creation

### Update Stripe webhook handler

(File name may vary, e.g. `/api/stripe/webhook.ts`)

#### Handle **ONLY** this event:

* `customer.subscription.created`

```ts
if (event.type === "customer.subscription.created") {
  const subscription = event.data.object as Stripe.Subscription;

  const customerId = subscription.customer as string;

  // Look up user by Stripe customer ID
  const user = await db.user.findFirst({
    where: { stripeCustomerId: customerId }
  });

  if (!user) {
    console.warn("No user found for Stripe customer", customerId);
    return;
  }

  // Idempotent update
  if (user.plan !== "pro_plus") {
    await db.user.update({
      where: { id: user.id },
      data: {
        plan: "pro_plus",
        planActivatedAt: new Date()
      }
    });
  }

  console.log("User upgraded to Pro+:", user.id);
}
```

### Guardrails

* ❌ Do NOT rely on checkout success redirect
* ❌ Do NOT update plan on client
* ✅ Webhook is the source of truth
* ✅ Idempotent (safe on retries)

## 3️⃣ Client: Invalidate User Profile Cache on Success

### In your main router or booking page effect:

```ts
import { useEffect } from "react";
import { useQueryClient } from "@tanstack/react-query";

const queryClient = useQueryClient();

useEffect(() => {
  const params = new URLSearchParams(window.location.search);
  const subscriptionStatus = params.get("subscription");

  if (subscriptionStatus === "success") {
    // Force refetch of user profile & capabilities
    queryClient.invalidateQueries(["userProfile"]);
    queryClient.invalidateQueries(["capabilities"]);

    // Optional: clean URL
    window.history.replaceState({}, "", "/booking-requests");
  }
}, []);
```

### Why this matters

* `hasCapability()` depends on `user.plan`
* Cached profile = stale entitlement
* This ensures **instant unlock**
