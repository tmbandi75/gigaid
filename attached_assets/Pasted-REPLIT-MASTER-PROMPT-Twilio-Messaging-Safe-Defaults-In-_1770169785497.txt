REPLIT MASTER PROMPT
Twilio Messaging + Safe Defaults + In-App Inbox + Usage Limits (GigAid)
OBJECTIVE

Enhance GigAid’s existing Send Text Message feature to support Twilio-based messaging while:

Preserving current copy-to-SMS behavior

Preventing missed client replies

Supporting a future-proof in-app inbox

Enforcing plan-based usage limits

Avoiding user confusion or trust loss

NON-NEGOTIABLE CONSTRAINTS

❌ Do NOT remove or alter existing copy-to-SMS flow
❌ Do NOT expose Twilio credentials to frontend
❌ Do NOT intercept or hide inbound replies by default
❌ Do NOT introduce “modes” language in UI
❌ Do NOT auto-upgrade users
❌ Do NOT silently drop messages

✅ Replies must reach the user’s phone unless explicitly opted out
✅ All enforcement happens server-side
✅ Free users must never miss replies

CORE DESIGN PRINCIPLE

If a user has to think about where replies go, the design has failed.

Default behavior must feel identical to native SMS.

PLAN-BASED USAGE LIMITS
🆓 FREE

Outbound via GigAid: limited (e.g. 25/month)

Inbound replies:

Always forwarded to phone

Limited forwarding count

In-app inbox: ❌ not available

💼 PRO+

Outbound via GigAid: higher limit (e.g. 500/month)

Inbound replies:

Stored in app if inbox enabled

Otherwise forwarded to phone

In-app inbox: ✅ available

Soft warnings at 80% usage

🏢 BUSINESS

Outbound via GigAid: high or unlimited

Inbound replies: unlimited

In-app inbox: ✅ available

Team inbox (future-safe)

OUTBOUND MESSAGE FLOW
API
POST /api/messages/send

Logic

Authenticate user

Verify entitlement + outbound limit

Send SMS via Twilio number

Log outbound message

Increment usage counter

Never block inbound due to outbound limits

INBOUND MESSAGE FLOW (TWILIO WEBHOOK)
API
POST /api/twilio/inbound

Logic
lookup user by twilio_number

if users.in_app_inbox_enabled AND plan >= Pro+:
  store message
  increment inbound_stored
else:
  forward message to users.personal_phone
  increment inbound_forwarded


Always return HTTP 200 to Twilio.
