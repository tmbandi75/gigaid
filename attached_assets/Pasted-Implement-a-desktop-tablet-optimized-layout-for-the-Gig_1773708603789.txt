Implement a desktop/tablet optimized layout for the GigAid Messages page while keeping the smartphone UI completely unchanged.

The goal is to convert the messages page into a professional messaging workspace for web users without modifying any existing logic or APIs.

CRITICAL RULES

1. DO NOT change the smartphone UI.
2. Mobile layout (<768px width) must render exactly the same as today.
3. DO NOT refactor existing messaging logic.
4. DO NOT modify backend APIs.
5. DO NOT rename props, hooks, or state variables.
6. DO NOT change message sending or receiving logic.
7. Only add UI enhancements for desktop/tablet layouts.
8. Reuse all existing conversation data and handlers.
9. Avoid modifying existing files except for responsive wrappers.

TECH STACK

React  
Tailwind CSS  
ShadCN UI  
Existing GigAid messaging components

RESPONSIVE RULES

Mobile (<768px)
Render the existing Messages page unchanged.

Desktop/Tablet (>=768px)
Render the new messaging workspace layout.

Use Tailwind visibility rules:

block md:hidden → existing mobile layout  
hidden md:block → new desktop/tablet layout

IMPLEMENTATION STRATEGY

1. Leave the existing Messages component untouched.

2. Create a new component:

3. In the Messages page file wrap rendering:

<div className="hidden md:block">
  <MessagesDesktopView />

4. Reuse the same conversation data already used by the mobile component.

5. Reuse the same sendMessage handler.

6. Do NOT introduce new APIs.
